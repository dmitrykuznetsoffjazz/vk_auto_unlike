# VK Unlike

[🇬🇧 English](README_EN.md) | [🇷🇺 Русский](README.md)

## Что делает

Автоматически снимает все реакции с постов в разделе «Реакции» ВКонтакте.  
Интерфейс с настройками позволяет управлять задержками и паузами прямо в окне браузера.

## Важно знать

### По работе скрипта:

* Скрипт работает только с открытым разделом [«Реакции»](https://vk.com/feed?section=likes). Скрипт открывает браузер, где необходимо пройти авторизацию, после чего перейти на страницу с реакциями (или нажать кнопку)

* Снимаются **все** реакции (лайки, злость, радость).

* Удаление реакций с удалённых постов **невозможно**.

* Встроен механизм автоматического прохождения простой капчи (чекбокс «Я не робот»). Если капча впоследствии усложнится (слайдер или картинки), скрипт приостановится и попросит пройти её вручную, после чего продолжит работу.

* При слишком частых действиях ВК может временно заблокировать возможность ставить реакции — скрипт делает вариативные паузы, чтобы этого избежать, но гарантий нет. Если увидели ошибку «Неизвестная ошибка», просто подождите и запустите снова, есть встроенная кнопка для паузы.

* Все настройки сохраняются в реальном времени, прогресс снятия реакций не теряется при остановке.

* Не сворачивайте окно браузера — скрипт должен оставаться видимым для корректной работы MutationObserver. Можно держать его на втором мониторе.

### По проекту:

* Используется **Puppeteer** для автоматизации браузера.

* **Chrome for Testing** автоматически загружается и обновляется при первом запуске (скрипт проверяет версию и загружает актуальную, если необходимо).

* Настройки (задержки, паузы) хранятся в `settings.json`, общий счётчик убранных реакций — в `totalLikes.json`.

* При каждом запуске открывается новая сессия браузера (cookies и история не сохраняются между запусками).

* При нажатии кнопки «Стоп» в панели управления скрипт завершится, браузер закроется, прогресс сохранится.

## Как пользоваться

### Требования для запуска:

* Установленный [Node.js](https://nodejs.org/en/download/) версии 18 или выше.

* Менеджер пакетов npm (входит в состав Node.js).

* Интернет для загрузки Chrome for Testing.

### Порядок действий:

1. Скачайте репозиторий.

2. Разархивируйте.

3. Откройте терминал в папке проекта и выполните:
	```bash
	npm install
	```
4. Дождитесь установки зависимостей (puppeteer и прочие).

5. Запустите скрипт командой:
	```bash
	npm start
	```

6. При первом запуске скрипт автоматически загрузит Chrome for Testing. Откроется окно браузера. Если потребуется авторизация — войдите в ВК вручную.

7. После загрузки страницы в правом нижнем углу появится кнопка «▶ Запустить скрипт». Нажмите её — появится панель управления и начнётся снятие реакций.

8. В левом нижнем углу отображаются текущие случайные значения задержки и лимита, а также статистика.

9. Если появится капча, скрипт попробует пройти её автоматически. В случае неудачи вы увидите сообщение «⛔ Пройдите капчу вручную» — просто выполните проверку в браузере, и работа возобновится.

10. Чтобы остановить скрипт, нажмите кнопку ⏹ в панели. Браузер закроется, все данные сохранятся, процесс в консоли завершится сам.

### Параметры настройки (через панель в браузере):

- ⏱ **Задержка (мс)** — базовая задержка между кликами (± разброс)
- 🔢 **Кликов до паузы** — через сколько реакций делать длинную паузу (± разброс)
- ⏳ **Длинная пауза (сек)** — продолжительность длинной паузы (± разброс)
- 🤖 **Капча (сек)** — задержка перед автоматическим кликом по чекбоксу капчи
- ⏲ **Ожидание кнопки (сек)** — максимальное время ожидания исчезновения нажатой кнопки

Все изменения сохраняются в `settings.json`.
